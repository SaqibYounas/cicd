Github Actions
CI (Continuous Integration) ka matlab hai code changes ko automatically test karna aur integrate karna, jisse development process fast aur bug-free hoti hai.

CD (Continuous Deployment) ka matlab hai ki code ko automatically production environment mein deploy kar dena jab tests pass ho jayein.

GitHub Actions

CI (Continuous Integration) means automatically testing and integrating code changes to make the development process faster and bug-free.

CD (Continuous Deployment) means automatically deploying the code to the production environment once all tests pass.







CI ka main purpose hota hai ke jab bhi koi developer code push kare, wo code automatically test ho jaye. Isse errors ya bugs time pe pakar liye jaate hain aur team ka kaam smooth chalta hai.
CD ka matlab hota hai ke agar sab tests pass ho jaayein, to code ko automatically production ya server pe deplo
y kar diya jaye, bina manual process ke.

// src/components/Counter.jsx
import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p data-testid="count-text">Count: {count}</p>
      <button onClick={() => setCount(count + 1)} data-testid="count-btn">
        Increase
      </button>
    </div>
  )
}



// src/components/Counter.test.jsx
import { render, screen, fireEvent } from '@testing-library/react'
import { describe, test, expect } from 'vitest'
import Counter from './Counter'

describe('Counter component', () => {
  test('renders initial count', () => {
    render(<Counter />)
    expect(screen.getByTestId('count-text').textContent).toBe('Count: 0')
  })

  test('increments count on button click', () => {
    render(<Counter />)
    const button = screen.getByTestId('count-btn')
    fireEvent.click(button)
    expect(screen.getByTestId('count-text').textContent).toBe('Count: 1')
  })
})

Agar aap npm run test chala rahe ho aur error aaye: "No test files found" ‚Äî to aap kaise debug karoge?


Ye error tab aata hai jab test runner ko koi test file nahi milti ‚Äî matlab ya to test files banayi nahi gayi, ya unka naming pattern test runner ke config ke mutabiq nahi hai.
Debug karne ke liye:

Pehle check karenge ke test files exist karti hain ya nahi.
Ensure karenge ke file names test runner ke include patterns (e.g., *.test.js, *.spec.ts) ke sath match karte hain.
Agar koi config file (like vitest.config.js ya jest.config.js) hai to usme exclude/include settings check karenge.
Finally test scripts package.json me sahi defined hon ya nahi.
