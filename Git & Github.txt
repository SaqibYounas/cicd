Untracked
Matlab:
Wo files jo abhi tak Git ke under track nahi ho rahi hain. Matlab, aapne nayi file banayi hai ya project mein add ki hai, lekin Git ko abhi ye nahi pata ki is file ko track karna hai ya nahi.

Kaise hoti hai:
Jab aap ek naye file ko project folder mein add karte hain, aur git add command nahi chalate, tab wo file untracked state mein hoti hai.

Action:
Agar aap is file ko Git repository mein track karna chahte hain to git add <filename> use karein. Agar nahi karna chahte, to file wahi rahegi.

2. Modified
Matlab:
Wo files jo Git ke under track ho rahi hain, lekin ab unme changes kar diye gaye hain. Matlab, Git ke paas file ki purani version hai, lekin ab usse different hai.

Kaise hoti hai:
Jab aap ek file ko edit karte hain (kuch naya add ya remove karte hain), tab wo file modified state mein aa jati hai.

Action:
Aap modified files ko Git ko batane ke liye git add <filename> chalate hain. Isse file staged state mein chali jaayegi.

3. Staged
Matlab:
Wo files jo commit ke liye ready hain. Matlab, aapne file ko modified state se utha kar Git ko bata diya hai ki "ye file ab commit ke liye ready hai".

Kaise hoti hai:
Jab aap git add <filename> chalate hain, tab wo file staged state mein chali jaati hai.

Action:
Aap staged files ko commit karne ke liye git commit -m "<message>" chalate hain.

4. Unmodified
Matlab:
Wo files jo Git mein tracked hain aur abhi unme koi changes nahi kiye gaye. Matlab, file waise hi hai jaise last commit ke baad thi.

Kaise hoti hai:
Jab aapne file ko commit kiya aur tab se us file mein koi bhi change nahi hua, to wo file unmodified state mein hoti hai.

Action:
Aapko unmodified files ke liye koi action lene ki zarurat nahi hoti, kyunki unka status waise hi rahta hai.

üîπ 1. origin kya hota hai?
By default jab aap repo clone karte ho, Git remote ka naam origin rakhta hai.

Example:
git clone https://github.com/username/project.git
Ab remote list dekhen:

git remote -v
Output:

origin  https://github.com/username/project.git (fetch)
origin  https://github.com/username/project.git (push)
Agar aap is remote connection ko delete karna chahein:

git remote remove origin
üîπ 2. github, upstream, ya custom naam
Kabhi kabhi aap manually ya kisi tutorial mein dusra remote add karte ho aur uska naam github, upstream, production, etc. rakh lete ho.

Example:
git remote add github https://github.com/username/project.git
Check remote:

git remote -v
Output:

github  https://github.com/username/project.git (fetch)
github  https://github.com/username/project.git (push)
Toh is remote ko delete karne ke liye:

git remote remove github

github	git remote remove github	Custom remote added manually
upstream	git remote remove upstream	Forked repos or secondary remotes

5. Git Remote Link Set Karen
Github repository ka remote link set karna hoga taake local changes remote pe upload ho sakein.


git remote add origin https://github.com/USERNAME/REPOSITORY_NAME.git
Yahan USERNAME ki jagah apna Github username aur REPOSITORY_NAME ki jagah apni repository ka naam likhen.

6. Files Ko Staging Mein Add Karen
Ab aapko apni files ko staging area mein add karna hoga taake unhe commit kiya ja sake.


git add .
Yeh command sari changes ko staging mein daal dega.

7. Changes Commit Karen
Ab apni changes ko commit karen:


git commit -m "Your commit message"
Yahan Your commit message mein aap wo likhen jo aapke changes ko explain kare.

Summary of Steps:
Create a Github account.
Create a new repository on Github.
Initialize Git in your local directory.
Set the remote repository URL.
Add your files using git add ..
Commit your changes using git commit -m "message".
Push the changes to Github using git push.
